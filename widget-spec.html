Save Widget

Component specification for designers and product managers

Overview

A floating banner that appears at the bottom-center of the viewport when a user modifies any form element on the page. It provides two actions: Reset (revert all changes) and Save Changes (persist and confirm). If the user tries to navigate away while the banner is visible, it shakes to signal that changes must be addressed first.

Careful , you have unsaved changes!

Interaction Behaviour

Appear
When any tracked field changes from its saved value
The widget slides up from below the viewport. It stays visible as long as there are unsaved changes. If the user manually reverts all fields to their original values, it slides back down automatically.
Reset
User clicks "Reset"
All fields revert to their last saved values. Any dynamically added rows or groups are removed. The widget slides down out of view.
Save
User clicks "Save Changes"
Current values become the new baseline. The banner morphs into a green toast message showing a checkmark and "Saved". After a brief hold, it slides down and resets internally.
Navigation Guard
User clicks any navigation element while the banner is visible
The widget shakes horizontally to draw attention. Navigation is blocked. The user must save or reset before leaving the page.
